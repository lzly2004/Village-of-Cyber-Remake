## 最终重构方案：分步执行计划

### 现有基础设施

| 设施 | 作用 |
|------|------|
| `BaselineRunner` | 回归测试运行器，7种配役固定种子 |
| `baseline/` | 黄金标准基线（7个文件） |
| `baseline_new/` | 新运行输出目录 |
| `GameRecorder` | 游戏事件录制器 |
| `ConstNum.setRandomSeed/startRandomRecording/...` | 随机数固定与校验 |
| `GameContext` | 已部分集成（`ctx` 字段，数组别名） |
| `logicTools` | 工具类（`log()`, `probabilityJudge()`） |

### 验证流程（每步重复）

```
1. 运行 BaselineRunner → 生成 baseline_new/
2. diff -r baseline/ baseline_new/ 
3. 除时间戳行外，所有事件、状态、随机数校验和必须完全一致
4. 不一致 → 回退修改，修复后重新验证
```
