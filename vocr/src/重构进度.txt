一、项目概述

  项目: 电脑村重置版 (VOCR)
  语言: Java
  类型: 人狼游戏（Werewolf/Mafia）文字模拟器
  核心类: MainLogic.java (3477行) — 重构的主要目标
  架构: Game(单例) -> UI / Resources / MainLogic 三模块

二、重构目标
────────────────────────────────────────────────────────────────────
  原始问题: MainLogic.java 过于庞大（3477行），所有游戏逻辑、状态管理、
           概率计算、怀疑度系统混在一起，难以维护和扩展。

  重构策略: 分阶段解耦
  Phase 1: 提取独立子系统（工具类、概率计算、怀疑度、常量）
  Phase 2: 创建 GameContext 统一状态管理（封装 GameStatus + 核心数组）
  Phase 3: 统一数组引用，消除 MainLogic 中的裸字段和 gs 直接访问
  Phase 4: 进一步拆分 MainLogic 方法到独立的 Handler 类
  Phase 5: 修复 Bug 清单（见"目前的目标.txt"）

五、关键设计决策
────────────────────────────────────────────────────────────────────
  1. GameContext 同时提供只读接口(GameContextView)和包级Action API
     — 子系统通过 GameContextView 查询，通过 Action API 修改
  2. MainLogic 本地数组指向 ctx 数组（而非删除本地数组）
     — 减少对现有代码的破坏，渐进式迁移
  3. ProbabilityCalculator 使用纯文本 config 文件
     — 从 JSON 配置改为 .txt，简化格式
  4. Game 类使用单例模式，管理 UI/Resources/MainLogic 三模块
